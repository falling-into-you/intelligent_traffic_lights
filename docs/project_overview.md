# 智能交通信号灯项目简介

本文整理自腾讯开悟智能交通信号灯调度开发指南，用于说明任务目标、场景元素和最终计分规则。

## 任务目标

本任务要求训练一个智能体控制单交叉口信号灯。智能体需要根据交通状态选择：

```text
信号灯相位 phase_idx
相位持续时间 duration
```

目标是在不同复杂度的交通环境中缓解交叉口及进出口道拥堵，提高车辆通行效率。

## 场景介绍

地图基于真实交叉口构建，为单个十字路口。智能体拥有全局视角，可以观测交叉口及附近交通参与者状态。

主要元素如下：

| 元素 | 说明 |
| --- | --- |
| 车辆 | 从既定车道生成，经过交叉口后从目的车道离开 |
| 进出口车道 | 车辆生成、汇入、离开的道路 |
| 交叉路口 | 车辆汇集、转向和疏散的位置 |
| 信号灯 | 智能体控制的实体，用于分配通行权 |
| 视野范围 | 全局视野，包含车辆长度、位置、车道、速度、角度等信息 |

代码中主要通过 `frame_state["vehicles"]` 读取车辆状态，并通过 `env.step(act)` 控制信号灯。

## 任务结束条件

任务可能正常结束，也可能超时结束。

| 类型 | 条件 |
| --- | --- |
| 正常结束 | 智能体预测步数达到配置的 `max_step` |
| 超时结束 | 等待车辆数达到 `max_waiting_cars`，且持续时间达到 `max_waiting_cars_duration` |

等待车辆指场景中速度为 0 的车辆，不包含事故车辆。

如果环境超时结束，本局得分为 0。

## 计分指标

任务结束后，环境根据四类指标计算总得分：

```text
车辆平均延误
车辆平均排队长度
车辆平均等待时间
信号变换频率
```

这些指标是环境评分标准，不等同于训练代码里的 `reward`。训练 reward 需要在 `reward_shaping()` 中自行设计，但应尽量和这些指标方向一致。

## 车辆平均延误

车辆延误表示车辆受到交通设施和交通流干扰后，相比自由流状态多消耗的时间。

单车延误：

```text
d = t_real - t_free
```

自由流行驶时间：

```text
t_free = s / v_max
```

含义：

| 符号 | 说明 |
| --- | --- |
| `d` | 单车延误 |
| `t_real` | 车辆实际已行驶时间 |
| `t_free` | 自由流状态下的行驶时间 |
| `s` | 车辆当前已行驶路程 |
| `v_max` | 车辆最大速度 |

环境会统计每步进口车道上车辆的平均延误，并对整局过程取平均。

## 车辆平均排队长度

排队长度统计进口车道上处于等待状态的车辆数量。

等待状态定义：

```text
车辆速度 <= 0.1 m/s
```

评价指标会对每个时刻的等待车辆数取平均：

```text
q_avg = 所有时刻排队车辆数的平均值
```

这个指标和当前基础 reward 里的 `queue_ratio` 方向一致：停车车辆越多，奖励越低。

## 车辆平均等待时间

等待时间表示车辆在进口车道上处于等待状态的累计时间。车辆变更交叉口后，等待时间会重置。

环境会统计每步进口车道上车辆的平均等待时间，并对整局过程取平均。

这个指标可以用于 reward 设计：

```text
等待时间下降 -> 奖励增加
等待时间上升 -> 奖励降低
```

当前 `target_dqn` 的基础 reward 已使用 `vehicle["waiting_time"]` 计算等待时间变化。

## 信号变换频率

信号变换频率用于衡量相位切换是否过于频繁。

计数规则：

```text
连续两个绿灯相位之间的时间间隔 < 8 秒
则信号变换频率计数 +1
```

这里的时间间隔包含中间黄灯持续时间。

如果模型为了降低等待时间频繁切换相位，最终得分会被扣减。因此后续 reward 可以加入切换惩罚，或在动作策略中限制过短 duration。

## 总得分公式

总得分由三个正向效率指标和一个切换惩罚组成：

```text
总得分 =
(车辆平均延误得分 + 车辆平均排队长度得分 + 车辆平均等待时间得分) * 1000
- 信号变换频率得分
```

各项得分：

```text
车辆平均延误得分 = 1 / (1 + 车辆平均延误 / 9)
车辆平均排队长度得分 = 1 / (1 + 车辆平均排队长度 / 10)
车辆平均等待时间得分 = 1 / (1 + 车辆平均等待时间 / 8)
信号变换频率得分 = 信号变换频率 * 1.5
```

结论：

```text
延误越低，得分越高
排队越短，得分越高
等待越少，得分越高
频繁切换越少，得分越高
```

## 对当前代码的启发

当前 `target_dqn` 的基础目标应先对齐环境得分方向：

| 环境指标 | 当前可用信号 | 代码位置 |
| --- | --- | --- |
| 平均等待时间 | `vehicle["waiting_time"]` | `agent_target_dqn/feature/definition.py` |
| 排队长度 | `vehicle["speed"] <= 0.1` | `agent_target_dqn/feature/definition.py` |
| 信号变换频率 | `act[1]`、`act[2]`、`phases` | 后续可增强 |
| 平均延误 | `vehicle["delay"]` | 后续可增强 |

第一版 reward 不需要覆盖全部评分规则。先让智能体学习减少等待和停车，再逐步加入延误、排队长度和切换惩罚。
